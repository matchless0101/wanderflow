## 目标
解决地点标注与实际位置不符的问题，统一坐标来源为系统内置地理编码，确保地图标注与路线规划坐标一致。

## 数据流
用户输入 → AI 生成行程（只含地点名称 + 顺序） → CLGeocoder 地理编码 → 标准坐标列表 → 路线规划 API → 地图 SDK 渲染。

## 关键改动
### 行程输出
AI 仅输出地点名称、顺序、时间与活动描述，不输出坐标与坐标系。

### 地理编码
客户端使用 CLGeocoder 将地点名称解析为坐标，顺序保持与行程一致。
解析结果缓存于内存，避免重复请求。
解析失败的地点不进入路线规划，保持行程显示但不参与标注与路线。

### 坐标系统
CLGeocoder 返回 WGS84 坐标。
地图渲染与高德路线使用 GCJ02，通过现有坐标转换逻辑统一处理。

## 错误处理
部分地点解析失败时提示“部分地点无法定位”，但仍展示可解析的标注与路线。
全部失败时清空路线与标注，仅保留当前位置。

## 验证策略
使用常见地点名进行手动验证，确认标注与路线一致。
构建项目以确保新流程无编译问题。
